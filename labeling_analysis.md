| 라벨 | 핵심 키워드 | 실제 예시 |
|------|-----------|----------|
| `EXTERNAL_TRADE` | 카톡, 010, 계좌, 입금, 수고비, 직거래, 연락처 | *"010구구칠팔06삼공"* / *"수고비 정두?"* / *"입근드린다했어여"* |

## 3. 공통 라벨 체계 & 분포

### 3-1. 확정 라벨 5종

| 라벨 코드 | 표시명 | 정의 |
|-----------|-------|------|
| `EXTERNAL_TRADE` | 외부거래 | 플랫폼 외부 거래 유도/시도 (연락처·계좌·카톡 교환, 시세 언급 포함) |
| `INAPPROPRIATE_CONTENT` | 부적절 콘텐츠 | 욕설, 모욕, 비하, 폭력적 위협, 딜러/컬렉터 공격 |
| `FRAUD` | 사기/부정 | 부계정, 개인정보 악용, 사기 행위 |
| `PRODUCT_SELLING` | 상품판매 | 허가되지 않은 상품 판매/공구 (현재 수집 데이터에서는 미검출) |
| `OTHER` | 기타 | 위 항목에 해당하지 않거나, 사유 미기입 건 |

### 3-3. 구분별 라벨 히트맵

```
                    EXTERNAL  INAPPROP  FRAUD  PRODUCT  OTHER
감지 (Live Chat)     ■■■       ■■■■■■■■  ·       ·        ·
```

### 3-4. 핵심 인사이트

1. **INAPPROPRIATE_CONTENT가 58%로 최다** — Live Chat 감지 + 신고에서 대부분. 단, 경고·제재로 이어지는 건은 0건
2. **EXTERNAL_TRADE가 28%** — 감지·신고·경고·제재 **전 단계에서 공통 출현**. 가장 중요한 추적 라벨
3. **OTHER가 13%로 신고에서 높음** — 신고 사유 중 "시세/가격 언급", "구매방해", "딜러 언급" 등이 기존 라벨에 매핑 안 됨. 라벨 세분화 검토 필요
4. **FRAUD는 경고·제재에서만 등장** — 감지·신고 단계에서는 부계정을 잡기 어려움 (별도 시스템: `#alert_user_abusing`)
5. **PRODUCT_SELLING은 0건** — 현재 patrol 채널에서 상품판매를 감지하는 구조가 아님 (`#alert_auto_patrol_product`에서 별도 관리)
6. **DM Patrol은 라벨링이 핵심 과제** — 1,817건 중 EXTERNAL_TRADE 비율 추정이 필요

## 4. DM Patrol 라벨링 기준안 (제안)

DM은 자동 라벨이 없으므로, 키워드 기반 자동 + 어드민 수동 검토 병행 권장.

### 자동 라벨링 키워드 규칙

| 라벨 | 키워드 (OR 조건) | 비고 |
|------|----------------|------|
| `EXTERNAL_TRADE` | `카톡`, `카카오`, `010`, `계좌`, `입금`, `수고비`, `직거래`, `연락처`, `전화`, `번호`, `은행`, `이체` | 1개 이상 매칭 시 |
| `FRAUD` | `주민`, `신분증`, `개인정보` | |
| `INAPPROPRIATE_CONTENT` | DM에서는 적용 안 함 (별도 맥락 필요) | |
| `OTHER` | 위 키워드 없음 | false positive 가능성 높음 → 어드민 검토 대상 |

### 예상 분류 결과 (추정)
- EXTERNAL_TRADE: ~40-50% (연락처/입금 관련 대화가 주 트리거)
- OTHER/미분류: ~50-60% (일반 대화가 키워드에 걸린 false positive)
